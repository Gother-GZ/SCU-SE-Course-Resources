## 共建计划

为了让每一份努力都能精准地帮到后来人，我们将贡献内容划分为以下几个板块。你可以根据手头的资料，选择最适合你的赛道：

#### 1️⃣ 资源分享：填补空白

本仓库致力于覆盖四川大学软件学院本科全阶段课程。你可以重点关注以下四类资源：

**🧠 核心复习类**

- **学霸笔记**：个人复习文档、手写笔记扫描件、知识点思维导图。
- **避坑指南**：课程难点解析、复习心得、公式速查表。
- *💡 建议格式：Markdown 最佳，PDF 次之。*

- **⚔️ 实战演练类**
  - **历年真题**：期中/期末试卷回忆版、小测题目。
  - **参考解析**：不仅要有题，最好能提供你的解题思路或标准答案。
- **💻 工程实践类**
  - **课设/大作业**：源代码、需求文档、实验报告。
  - **平时作业**：课堂练习、小组作业成果。
  - *💡 建议：代码类请附带简易 README 说明运行环境。*
- **📚 辅助教材类**
  - **教材指引**：推荐教材的名称/ISBN（一般不直传电子书）。
  - **思政教辅**：思修、马原等课程的习题集或重点整理。

------

#### 2️⃣ 提交红线：避坑指南 (Restrictions)

在上传上述资源时，请务必遵守以下“红绿灯”规则，避免做无用功：

- 🔴 **严禁上传**
  - **教师课件**：PPT/Slide 属于老师知识产权，未经授权严禁上传。
  - **版权限制**：明确声明禁止转载的第三方资源。
- 🟡 **特殊处理**
  - **超大文件 (>100MB)**：请上传至网盘，并在仓库中提交链接文件。
  - **视频资源**：请**仅提供在线链接**，不要直接上传视频文件。

------

#### 3️⃣ 社区维护：持续优化 (Maintenance)

除了上传新资源，对现有内容的**维护**与**思考**同样珍贵：

- **🛠️ 园丁模式**
  - **捉虫修补**：发现 Typo（拼写错误）或排版混乱？直接 PR 修复！
  - **时效更新**：考点变了？技术栈更新了？请用新版资料替换旧版。
  - **多维补充**：同一门课，如果你有不同的解法或笔记，欢迎提交“Ver.2”版本。
- **🗣️ 参谋模式**
  - 觉得目录结构不合理？有想要增加的功能？请移步 **Issues** 区畅所欲言。
- **🤝 深度参与**
  - 如果你愿意长期参与审核 PR、整理文档，欢迎联系我们，成为核心贡献者！















## 2. 贡献规范 📋

为了保持整洁和方便检索，请严格按照规范进行贡献资料 (｡•̀ᴗ-)✧

### 🚫 需要避免的问题

**请务必不要上传以下内容** ❌：

1. **涉及隐私的内容**：电话号码、家庭住址、身份证号等个人信息 🔒
2. **主观攻击性内容**：对老师或课程的纯粹情绪化吐槽（客观评价是可以的）😤
3. **版权侵犯内容**：**严格禁止上传任何老师的课件**（PPT/讲义/电子书等）⚠️
4. **违规违法内容**：详见[《负面清单》](./NEGATIVE_LIST.md) 📜

### 📁 文件命名规范

我们实行 **"文件夹一级命名负责制"**。简单来说：

> **我们只强制要求你上传的根文件夹（第一层）遵循规范** ✅  
> 文件夹内部的结构，你可以根据项目实际情况自由安排，但请做到 **"见文知意"**！

**命名格式：**

```text
入学年份_课程名称_作品类型_作者或描述
```

**各部分说明：**
- **入学年份**：你入学的年份（4位数字），如 `2021`、`2022`
- **课程名称**：使用简洁易懂的名称，如 `OperatingSystem`、`Java`、`DataStructure`
- **作品类型**：`Lab`（实验）、`Proj`（课设/项目）、`Notes`（笔记）、`Exam`（试题）等
- **作者或描述**：可以是你的昵称（脱敏后），或者对作品的简短描述

1. **层次清晰**：不要把所有文件堆在一个文件夹里 🗂️
2. **命名规范**：文件名要有意义，避免 `新建文本文档.txt`、`未命名1.docx` 这样的名字 📝
3. **附带说明**：如果项目稍复杂，请写一个 `README.md`，说明如何使用 📖

---

### ✅ 贡献前的检查清单

在提交资料前，请花 1 分钟检查以下几点：

- [ ] **合规合法**：资料是否合规合法，没有违反[负面清单](./NEGATIVE_LIST.md) ✔️
- [ ] **隐私脱敏**：是否已去除敏感个人信息（姓名、学号、电话等）🔐
- [ ] **避免重复**：检查课程文件夹下是否已有类似资料，避免重复上传 🔍
- [ ] **文件完整**：代码能运行？文档能打开？链接有效？✨

**全部打钩？那就放心提交吧！** (ง •̀_•́)ง

#### � PR 提交完整教程

**整体流程：**
```
Fork → Clone → Add Files → Commit → Push → Pull Request → Review → Merge
```

**详细步骤：**

##### 1️⃣ Fork 本仓库 🍴

1. 登录你的 GitHub 账号（如果还没有，请先去 [GitHub](https://github.com) 注册）
2. 进入 [本仓库首页](https://github.com/YourRepo/SCU-SE-Course-Resources)
3. 点击右上角的 **Fork** 按钮
4. 确认 Fork，稍等片刻，你的账号下就有了一个同名的仓库副本 ✨

之后你会看到仓库名下方显示：`forked from YourRepo/SCU-SE-Course-Resources`

---

##### 2️⃣ Clone 到本地 💻

打开你的终端（Windows 用 Git Bash，Mac/Linux 用 Terminal），执行：

```bash
git clone https://github.com/你的用户名/SCU-SE-Course-Resources.git
cd SCU-SE-Course-Resources
```

这样，仓库就被下载到你的本地电脑了 📥

---

##### 3️⃣ 添加你的资源 📂

1. **找到对应的课程目录**，比如 `必修/02_大一下/Java程序设计/`
2. **创建你的文件夹**，记得遵守命名规范！比如：
   ```
   2022_Java_Proj_SpaceInvaders_YourName/
   ```
3. **把你的文件放进去**：代码、文档、README……
4. **最后检查一遍**：
   - 有没有个人隐私信息？🔒
   - 代码能正常运行吗？▶️
   - 有没有违规内容？⚠️

---

##### 4️⃣ Commit 提交 💾

回到终端，执行以下命令：

```bash
# 查看修改了哪些文件
git status

# 添加所有修改到暂存区
git add .

# 提交修改，并写上清晰的提交信息
git commit -m "feat: add 2022 Java SpaceInvaders project"
```

**关于 Commit Message：**
- 建议使用格式：`类型: 简短描述`
- 常用类型：`feat`（新增）、`fix`（修复）、`docs`（文档）、`refactor`（重构）

---

##### 5️⃣ Push 到远程仓库 🚀

```bash
git push origin main
```

（如果你创建了新分支，把 `main` 替换成你的分支名）

现在，你的修改已经上传到**你自己 Fork 的仓库**了 ☁️

---

##### 6️⃣ 发起 Pull Request 🔀

1. 回到 GitHub 页面，进入你 Fork 的仓库
2. 你会看到一个黄色提示条，上面写着 **"Compare & pull request"**，点击它
3. **填写 PR 信息：**
   - **标题**：简洁明了，如 `Add 2022 Java SpaceInvaders Project`
   - **描述**：简单介绍你上传了什么，有什么特点
4. 点击 **"Create pull request"**

恭喜！🎉 你的 PR 已经提交，接下来就等待仓库维护者审核了 (ﾉ◕ヮ◕)ﾉ*:･ﾟ✧

---

##### 7️⃣ 等待 Review 👀

维护者会检查你的 PR：
- 是否符合命名规范？✅
- 有没有违规内容？⚠️
- 文件是否完整？📦

如果有问题，维护者会留下评论，你只需要按照建议修改后再次 Push，PR 会自动更新 🔄

如果一切顺利，你的 PR 会被 **Merge（合并）** 到主仓库，你就正式成为了 **Contributor** 之一！🌟🌟🌟

#### � 常见问题 FAQ

**Q: 我不会用 Git Bash，可以用网页版操作吗？**  
A: 可以的！你可以直接在 GitHub 网页上进行操作：

A: 尝试搜索 "Git 解决冲突"，或者在 Issue 中求助，我们会帮你 (•̀ω•́)✧

**Q: 提交 PR 后多久会被审核？**  
A: 通常在 3-7 天内，请耐心等待。如果超过 7 天没有回复，可以在 PR 下留言提醒 �
